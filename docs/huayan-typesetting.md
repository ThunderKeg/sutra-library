# 《大方廣佛華嚴經》内容与排版规范

## 1. 适用范围

本规范约束《大方廣佛華嚴經》汉语拼音版在藏书阁中的正文、拼音、页序、版面和阅读交互。当前实现范围是第一册，后续各册应沿用同一原则。

## 2. 原始依据与保真原则

- 以圓道禪院发布的《大方廣佛華嚴經》汉语拼音版 PDF 为最高依据。
- 正文必须保持原书繁体字，不得自动简化、现代化、润色、补写或删节。
- 经名、品名、卷次、译者署名、偈颂、咒文、标点和正文顺序应与原书一致。
- 读者可见页面只保留原书内容、必要的结构标题、译者信息和文本来源，不加入生成方法、核验过程或内部页码映射说明。
- 原始来源统一标注为“文本来源 · 圓道禪院漢語拼音版”，并链接到公开下载页。

## 3. 正文排版

- 使用 `writing-mode: vertical-rl`，每栏从上到下阅读，各栏从右向左排列。
- 正文字符、标点和拼音的顺序必须依据 PDF 坐标重建，不得使用横排文本顺序直接换行模拟。
- 保留原书的页级边界。每页必须拥有稳定的 `pageKey`、来源页标签及可恢复的页内位置。
- 卷首、章节标题和正文应保持清晰层级，但不得改变经文顺序。
- 字号可由读者调节，调整后仍需保持竖排、拼音对齐和当前阅读位置。
- 宣纸、护眼和夜读主题只能改变配色与对比度，不得改变正文内容或页序。

## 4. 汉语拼音

- 每个原书带注音的汉字使用 `<ruby>` 与 `<rt>` 逐字呈现声调拼音。
- 拼音以原 PDF 嵌入字形的实际形状为依据。`pypinyin` 只能帮助生成器寻找解码候选，不得用词典自动注音覆盖原书注音。
- 自定义拼音字形存在歧义时，必须对照原页字形核验，并将确定结果记录在 `scripts/huayan_pinyin_overrides.json`。
- 拼音必须保留声调符号、轻声和原书读音；不得擅自采用常见读音替换经文读音。
- 读者可以隐藏拼音；隐藏后正文、标点、页序和续读位置不得变化。

## 5. 复杂页面与影印保真

- 普通经文页应转换为可缩放、可选择的 HTML 文本。
- 前置资料、悉昙字形、华严字母表、附录、版记以及无法可靠恢复文字和拼音对应关系的页面，使用原页 2 倍尺寸无损 WebP 影印图。
- 不得为了提高 HTML 页比例而用缺字、错序或错误拼音替代影印页。
- 第一册当前固定为 624 个 PDF 页：555 个 HTML 文本页和 69 个影印页。调整这一划分必须有逐页证据，并同步更新生成器和检查基线。

## 6. 章节、目录与阅读位置

- 第一册目录包含前置资料、开经偈和卷一至卷十，共 13 个阅读章节。
- 目录标题、章节 ID 和页键是续读链接与本地进度的稳定接口；除非提供迁移逻辑，否则不得修改。
- 续读必须恢复到具体章节、具体原书页和页内相对位置。
- 阅读方向为从右向左，因此“下一节”位于左侧，“上一节”位于右侧。
- 到达章节边界后，继续向阅读方向拉动可进入下一节；在章节开头反向拉动可返回上一节末页。

## 7. 加载与离线阅读

- 首次打开只请求册次索引和当前章节，不得自动下载其他章节或整册影印资源。
- 整册离线缓存只能由读者点击“下载整册离线阅读”后开始。
- 下载应逐项执行并显示完成进度；中断后重试时复用已缓存资源。
- 已主动下载的经书使用独立离线缓存，普通应用界面更新不得将其删除。

## 8. 生成与复核

正文或拼音变更必须通过生成器产生，不能直接修改生成后的章节 JSON：

```powershell
python scripts/build_huayan.py
python scripts/check_site.py
python scripts/audit_huayan.py
```

复核至少确认以下事项：

- 原始 PDF SHA-256 和页数未发生意外变化。
- 每个 PDF 页在生成结果中恰好出现一次，页序连续。
- HTML 正文字符、拼音数量和 41 个缺失文字层字符的恢复结果符合基线。
- 独立审计器使用与生成器不同的提取路径核对全文。
- 影印页文件存在、尺寸正确且使用无损 WebP。
- 读者可见文本中没有技术生成说明或内部核验文案。

## 9. 诵经音频

- 当前采用慧平法师读诵的《八十华严》，来源页为佛弟子文库[《八十华严（慧平法师）》](https://www.fodizi.net/fojing/23/7444.html)。
- 音频文件由来源站点 CDN `https://wz.yyxcfg.com/a/a/4/` 在线串流；本项目不保存音频文件，也不将其加入应用壳或整册离线缓存。
- 首页统一标注文本与音频来源；阅读器只保留右上角“听／停”控制及必要的情境操作，不重复展示来源说明。
- 第一册前置资料与开经偈没有对应音轨，应明确显示不可播放，不得错配正文音频。
- 第一册页级时间点保存在独立的 `data/huayan/volume-01-audio-alignment.json`；它只能引用稳定的章节 ID、`pageKey` 与页码，不得把对齐结果写回章节正文 JSON。
- 对齐必须先按音轨实测时长、各页有效汉字数和平均语速生成粗估，再使用带词级时间戳的语音识别结果与校本文字做全局编辑距离匹配，并以同音字匹配补充单调时间锚点；原有繁体正文始终是校本，识别文本不得用于改写正文。
- 逐页进度以 `docs/huayan-audio-alignment-progress.md` 为准；生成后还须用独立检查器复核 614 页完整性、时间单调性、音轨范围、低置信度页和卷七跨章节连续性。
- 阅读位置是阅读与音频共用的唯一进度，不得另存或恢复独立的音频进度。重新进入阅读器后先恢复具体章节、原书页和页内位置；点击“听”时，再由该位置换算音频时间。
- 右上角“听”是直接播放按钮，不打开播放器面板，也不展示原生音频进度条。未播放时点击即从当前阅读位置开始，播放时按钮变为“停”，再次点击即暂停。
- 播放时依据页级时间边界连续、平滑地滚动经文，不做逐句或逐字高亮，也不得在相邻页之间反复硬跳。播放中手动翻页只暂停文字跟随，音频继续播放，并仅提供“回到播放处”“从这里播放”两个情境操作；暂停状态下翻页不得出现音频操作提示。
- 卷七跨章节共用同一音轨；自动跟随跨越章节边界时，应保留当前正文直至新章节就绪，并用轻量过渡遮蔽内容替换，不能先清空为加载占位而造成闪跳。
